# AMS UX Journey — Associate Lifecycle

## Philosophy

> **Don't design pages. Design the journey of an Associate from day one to becoming an active professional.**

### Traditional HRIS vs AMS

```
HRIS (Data-centric):
  Profile → Data → Documents

AMS (Cycle-centric):
  Join → Build → Match → Deliver → Review → Improve → Repeat
```

### Core Insight

**Assignment is the center of Associate activity**, not Dashboard.

The most frequent flow:
```
Assignment → Submit → Review → Capability Update → New Assignment
```

---

## The 8 Phases

```
1. DISCOVER    — Associate learns about BinaHub
2. JOIN        — Register and verify
3. ONBOARD     — Build professional profile (MOST IMPORTANT)
4. DASHBOARD   — Workspace (passed through once)
5. WORK        — Assignments and tasks (MOST USED)
6. REVIEW      — Feedback and capability update
7. GROW        — Development and learning
8. MAINTAIN    — Profile and document maintenance
```

## Priority Order for Implementation

1. **Onboarding** — First impression, most critical
2. **Work** — Core activity, most frequently used
3. **Review** — Feedback loop, drives growth
4. **Dashboard** — Quick overview, passed through
5. **Profile** — Professional identity
6. **Development** — Long-term growth
7. **Documents** — Maintenance
8. **Settings** — Configuration
